# Definition of done

## DoD list

### 1. Workflow behavior

The repository workflow explicitly requires a short post-run reflection on reporting/process learnings.

- [x] 1.1 The required workflow includes a step that checks whether the just-completed run revealed reusable reporting or workflow guidance worth codifying.
- [x] 1.2 That step says to update the relevant guidance file when a concrete reusable lesson is found, rather than leaving the reflection ad hoc.

### 2. Verification

The new reflection requirement is documented clearly and checked in the final report.

- [x] 2.1 The wording identifies where the update should land when needed, such as `.agents/reporting/dod-points.md` or the workflow source itself.
- [x] 2.2 Validation confirms the new reflection requirement is present in the workflow, and the final report distinguishes observed file edits from inference about future compliance.

## Undefined

- Whether the reflection should become a separate prompt file or remain inline in the workflow can stay unchanged unless implementation makes that necessary.
